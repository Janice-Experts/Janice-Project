# Tariff Code Browser — Design

**Date:** 2026-04-05

## Goal

Add a browsable tariff code tree with keyword-based categorization, accessible as a third tab on /search and as a modal from the tariff 🔍 button in the review page.

## Decisions

| Aspect | Decision |
|--------|----------|
| Categorization | Script scans tariff descriptions for keywords, assigns each code to one of ~15 categories |
| Data | New `public/data/tariff-hierarchy.json` generated by `scripts/generate-tariff-hierarchy.mjs` |
| API | Two new endpoints: `GET /api/tariff/hierarchy`, `GET /api/tariff/category?name=X` |
| Component | New `TariffCodeBrowser` — same UX pattern as ICDCodeBrowser but separate implementation |
| Search | Uses existing `/api/search/tariff` endpoint |
| Modal | ClaimGroupCard + ClaimCard: tariff 🔍 opens `TariffCodeBrowser` instead of `CodeSearchModal` |
| Standalone | Third tab on /search: "ICD-10 Codes" | "External Cause Codes" | "Tariff Codes" |
| Fallback | Codes that don't match any keyword go into "Other Procedures" |
| DB path | Static JSON now → `tariff_categories` table later |

## Categorization Script

`scripts/generate-tariff-hierarchy.mjs` reads `public/data/tariff-codes.json` and assigns each code to a category based on keyword matching against the description.

### Categories

| Category | Keywords (in description) |
|----------|--------------------------|
| General Rules & Modifiers | codes 0001-0009 (rule/modifier codes) |
| Consultations | consult, visit, assessment, counselling |
| Surgical Procedures | excision, incision, repair, resection, ligation, suture, drainage, debridement, exploration |
| Orthopaedics & Fractures | fracture, arthro, ortho, bone, joint, tendon, ligament, amputation, osteotomy, spine, vertebr |
| Anaesthesia | anaesth |
| Radiology & Imaging | x-ray, radiograph, CT scan, MRI, fluoroscop, mammograph, angiograph, ultrasound, sonograph, contrast, imaging |
| Pathology & Laboratory | culture, assay, antigen, antibod, haemoglobin, glucose, protein, cholesterol, enzyme, biopsy, histol, cytol, PCR, serology, haematol, chromatograph |
| Audiology & Speech | audiolog, hearing, cochlear, tympan, vestibul, speech, language therap |
| Ophthalmology & Optometry | ophthalm, optomet, retina, cataract, glaucoma, cornea, vitrect, lens, spectacle, visual field |
| Obstetrics & Gynaecology | obstet, gynae, pregnan, foet, caesarean, hysterect, cervix, uterus, vaginal, labia |
| Urology | urolog, ureth, bladder, cystect, prostat, renal, kidney, nephre, dialysis, catheter |
| Neurosurgery | neurosurg, craniotom, laminectom, spinal cord, intracranial, cerebr |
| Cardiothoracic | cardio, cardiac, coronary, bypass, valve, pacemaker, thoracot, lung, pleura, mediast |
| Allied Health | physio, biokinet, occupational therap, dietetic, nutrition, podiatr, chiropract |
| Materials & Medication | material, medication, medicine, dispensing, drug, implant cost, prosthetic device |
| Other Procedures | (fallback for unmatched codes) |

Keywords are matched case-insensitively. First matching category wins (order matters — more specific categories checked first). The script reports uncategorized codes so categories can be refined.

### Output: tariff-hierarchy.json

```json
[
  {
    "category": "Consultations",
    "codeCount": 105,
    "subcategories": [
      {
        "name": "General Consultations",
        "codes": [
          { "code": "0190", "description": "..." }
        ]
      }
    ]
  }
]
```

If a category has ≥30 codes, group them into subcategories (by first 2 digits of code). If <30, flat list.

## API Endpoints

**`GET /api/tariff/hierarchy`** — returns full category list with code counts.

**`GET /api/tariff/category?name=Consultations`** — returns all codes in a category:
- If <30 codes: flat array sorted by code
- If ≥30 codes: grouped into subcategories by 2-digit prefix, each sorted by code

Response shape mirrors `/api/icd/block` pattern.

## TariffCodeBrowser Component

Same UX as ICDCodeBrowser:
- Browse mode: category → subcategory (if grouped) → codes
- Search mode: split view with `/api/search/tariff` results (left) + tree (right)
- Modal mode: fixed overlay, max-w-4xl, "Tariff Code Browser" header
- Standalone mode: white card, no overlay
- Position memory (module-level, modal only)

Props:
```ts
interface TariffCodeBrowserProps {
  mode: 'modal' | 'standalone';
  onSelect: (code: string, description: string) => void;
  fieldId?: string;
  onClose?: () => void;
}
```

## Integration

### /search page

Third tab: "ICD-10 Codes" | "External Cause Codes" | "Tariff Codes"

When "Tariff Codes" active, render `TariffCodeBrowser` in standalone mode.

### Review page modals

ClaimGroupCard + ClaimCard: tariff 🔍 button opens `TariffCodeBrowser` in modal mode instead of `CodeSearchModal`.

`CodeSearchModal` will no longer be used for any code type after this change (ICD → ICDCodeBrowser, ECC → ICDCodeBrowser with codeFilter=ecc, Tariff → TariffCodeBrowser).

## Out of Scope

- Validation engine changes
- New tariff data sources
- Discipline-aware tariff validation
- Database persistence (future)
