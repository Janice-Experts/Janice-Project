## Goal

Add a hierarchical ICD-10 browser that lets users drill down by chapter and block to find the right code. Serves two contexts: as a lookup tool during claim corrections (modal) and as a standalone search page.

## Data Layer

### icd10-hierarchy.json

Static file at `public/data/icd10-hierarchy.json`:

```json
[
  {
    "chapter": "I",
    "title": "Certain infectious and parasitic diseases",
    "range": "A00-B99",
    "codeCount": 1247,
    "blocks": [
      { "range": "A00-A09", "title": "Intestinal infectious diseases", "codeCount": 47 },
      { "range": "A15-A19", "title": "Tuberculosis", "codeCount": 32 }
    ]
  }
]
```

- Generated by `scripts/generate-hierarchy.mjs` from `icd10-mit.json`
- Only includes blocks with â‰Ą1 SA-approved code
- Standard ICD-10 chapter/block definitions hardcoded in script (universal across countries)
- DB migration path: becomes `icd_chapters` and `icd_blocks` tables

### API Endpoints

**`GET /api/icd/hierarchy`** â€” returns full hierarchy (chapters + blocks + code counts). Cached server-side.

**`GET /api/icd/block?range=A00-A09`** â€” returns all SA codes in the block:
- If <15 codes: flat array sorted by code
- If â‰Ą15 codes: grouped by 3-digit category, each group sorted by code

Response shape:
```json
{
  "block": "A00-A09",
  "title": "Intestinal infectious diseases",
  "grouped": false,
  "codes": [{ "code": "A00.0", "description": "..." }]
}
```

Or when grouped:
```json
{
  "block": "I20-I25",
  "title": "Ischaemic heart diseases",
  "grouped": true,
  "categories": [
    {
      "category": "I20",
      "title": "Angina pectoris",
      "codes": [{ "code": "I20.0", "description": "..." }]
    }
  ]
}
```

### Layout Behaviour

**Search bar empty â†’ browse mode:**
- Full-width collapsible tree
- Chapter rows show title + code range + count
- Click to expand â†’ block rows
- Click block â†’ code list (flat or category-grouped based on count)
- Click code â†’ triggers onSelect

**Search bar has text â†’ search mode:**
- Left panel (~50%): search results from existing `/api/search/icd`
- Each result shows code, description, breadcrumb (chapter â€ş block)
- Right panel (~50%): tree browser
- Click a search result â†’ tree auto-expands to that code's chapter/block, highlights the code
- Can select from either panel

### Position Memory

Stored in React state (component-level, not persisted):
- `lastFieldId`: which field last opened the browser
- `lastBrowseState`: expanded chapters, expanded blocks, scroll position
- On open: if `fieldId === lastFieldId`, restore state. Otherwise reset.
- Cleared on page reload (React state).
